# レビュー観点

Pull Request をレビューするときは、相手を責めるのではなく、研究紹介を一緒に良くするつもりで確認します。

## 確認すること

- PR 本文に変更内容が書かれているか
- 変更理由が読み取れるか
- 変更範囲が自分の研究紹介ファイルだけになっているか
- 研究テーマが短く説明されているか
- 研究の目的が専門外の人にも分かるか
- 使うデータや方法が簡単に書かれているか
- 不要なファイルや個人情報が含まれていないか

## コメント例

```text
変更理由が PR 本文に書かれていて、意図が分かりやすいです。
```

```text
研究の目的が最初に書かれていて、専門外の人にも読みやすいです。
```

```text
「使うデータ」に、データの期間や単位が1文あると、さらに分かりやすくなりそうです。
```

```text
不要なファイルは含まれていないことを確認しました。
```

## 避けたいコメント

```text
よく分かりません。
```

理由がないコメントでは、相手が直し方を判断しにくくなります。気になった場所と理由を一緒に書きます。
